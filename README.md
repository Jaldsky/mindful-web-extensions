## üì• Installing the latest release (for users)

Install the extension from the official release package. No developer tools or command line required.

### English

1. **Open the releases page**
   - Go to: [Releases ¬∑ Mindful Web Extensions](https://github.com/Jaldsky/mindful-web-extensions/releases) (or your repo‚Äôs **Releases** tab).
   - Find the latest release (e.g. **v1.3.2**).

2. **Download the extension archive**
   - Under **Assets**, click **`chrome-v1.3.2.zip`** (the name includes the version number).
   - The file will download to your computer (usually to the **Downloads** folder).

3. **Unzip the archive**
   - Locate the downloaded file (e.g. `chrome-v1.3.2.zip`).
   - **Double‚Äëclick** it to open, or right‚Äëclick ‚Üí **Open**.
   - You should see a **folder** inside (e.g. `chrome-v1.3.2`). **Extract / unzip** that folder to a place you‚Äôll remember (e.g. **Desktop** or **Documents**).
   - On Windows: right‚Äëclick the zip ‚Üí **Extract All** ‚Üí choose a folder.
   - On macOS: double‚Äëclick the zip; a folder with the same name will appear next to it.

4. **Open Chrome‚Äôs extensions page**
   - Open **Google Chrome**.
   - In the address bar type: `chrome://extensions` and press **Enter**.

5. **Enable Developer mode**
   - On the **extensions** page, find the switch in the top‚Äëright named **Developer mode** and turn it **ON**.

6. **Load the extension**
   - Click the **Load unpacked** button (top‚Äëleft area).
   - In the window that opens, select the **folder** you unzipped (e.g. `chrome-v1.3.2`), then click **Select folder** (or **Open**).
   - The Mindful Web extension should appear in the list and its icon in the Chrome toolbar.

7. **Pin the icon (optional)**
   - Click the **puzzle piece** icon in the toolbar ‚Üí find **Mindful Web** ‚Üí click the **pin** icon so the extension is always visible.

You‚Äôre done. Click the extension icon to open it and sign in or use it anonymously.

## üöÄ Quick Start (developers)

### 1. Prerequisites
- Chrome 88+ with Manifest V3
- FastAPI backend running on `http://localhost:8000`
- Web frontend running on `http://localhost:5173` (or your configured URL)

### 2. Build & Install
```bash
cd extensions/chrome
npm install
npm run build
```

### 3. First Launch
1. Click extension icon
2. Choose **"Try Without Login"** (anonymous) or **"Login"** (with account)
3. Open the web frontend ‚Äî you will be **automatically** in the same auth state (anonymous or logged in) thanks to shared cookies
4. Configure settings, manage domain exceptions
5. Start tracking!

## üîí Privacy & Security

- **Secure Auth** ‚Äî HttpOnly cookie-based sessions; access/anonymous
- **SSO** ‚Äî Sign in once in the extension or on the web ‚Äî you're automatically signed in on the other
- **Local-First** ‚Äî All data stored locally before sending
- **Anonymous Mode** ‚Äî Use without account
- **Domain Only** ‚Äî Never tracks full URLs or content
- **User Control** ‚Äî Enable/disable tracking, manage exceptions
- **Minimal Permissions** ‚Äî Only necessary browser permissions

## üìù Data Format

Events sent to backend:
```json
{
  "data": [
    {
      "event": "active",
      "domain": "example.com",
      "timestamp": "2024-01-15T10:30:00.000Z"
    }
  ]
}
```

**Event Types:** `active`, `inactive`

**Batch Processing:** Every 30 seconds, offline queue, automatic retry with limits
